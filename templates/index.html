Understanding Vulnerabilities

Vulnerabilities are weaknesses in a system that give threats the opportunity to compromise an individual's or an organisation's assets. As the attackers are leveling up their attacking mechanisms, the number of vulnerable assets is increasing too. These could include the code, human weaknesses, unpatched software, improper authentication mechanisms, etc.

💻
Software Vulnerabilities
Bugs, buffer overflows, SQL injection, unpatched CVEs in applications and OS.
🔐
Authentication Flaws
Weak passwords, broken auth mechanisms, missing MFA, session hijacking risks.
🌐
Network Weaknesses
Missing SSL, exposed ports, misconfigured headers, unencrypted transmissions.
👤
Human Factor
Social engineering, phishing susceptibility, insider threats, poor security hygiene.
🛡
Configuration Errors
Default credentials, open S3 buckets, misconfigured firewalls, excessive permissions.
⚡
Zero-Day Exploits
Unknown vulnerabilities actively exploited before vendors release patches.
